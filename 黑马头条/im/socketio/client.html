客户端

Socket.IO Client包含两个Socket.IO客户端：

  • 在socketio.Client()类创建与标准Python库兼容的客户端。
  • 在socketio.AsyncClient()类创建与兼容的客户端asyncio软件包。

两个客户端中的方法是相同的，唯一的区别是在asyncio客户端中大多数方法都是作为协同程序实现的。

安装

要安装标准Python客户端及其依赖项，请使用以下命令：

pip install "python-socketio[client]"

创建客户端实例

要实例化Socket.IO客户端，只需创建相应客户端类的实例：

import socketio

# standard Python
sio = socketio.Client()

定义事件处理程序

Socket.IO协议是基于事件的。当服务器想要与客户端通信时，它会发出一个事件。每个事件都有一个名称和一个参数列表。客户端使用socketio.Client.event()或socketio.Client.on()装饰器注册事件处理函数：

@sio.event
def message(data):
    print('I received a message!')

@sio.on('my message')
def on_message(data):
    print('I received a message!')

在第一个示例中，事件名称是从处理程序函数的名称获取的。第二个示例稍微详细一些，但它允许事件名称与函数名称不同，或者包含在函数名称中非法的字符，例如空格。

在connect和disconnect事件是特殊的;当客户端连接或断开与服务器的连接时，它们会自动调用：

@sio.event
def connect():
    print("I'm connected!")

@sio.event
def disconnect():
    print("I'm disconnected!")

请注意，disconnect为应用程序启动的断开连接，服务器启动的断开连接或意外断开连接（例如，由于网络故障）调用处理程序。在意外断开连接的情况下，客户端将在调用断开处理程序后立即尝试重新连接。一旦重新建立连接，将再次调用连接处理程序。

如果服务器包含带有事件的参数，那么它们将作为参数传递给处理函数。

连接服务器

通过调用connect()方法建立与服务器的连接：

sio.connect('http://localhost:5000')

在连接时，服务器为客户端分配唯一的会话标识符。应用程序可以在sid属性中找到此标识符：

print('my sid is', sio.sid)

发送事件

客户端可以使用以下emit()方法向服务器发出事件：

sio.emit('my message', {'foo': 'bar'})

提供给该方法的单个参数是传递给服务器的数据。该数据可以是类型的str，bytes，dict，list或tuple。发送a时tuple，其中的元素必须是其他四种允许类型中的任何一种。元组的元素将作为多个参数传递给服务器端事件处理函数。

emit()可以在事件处理程序内调用该方法作为对服务器事件的响应，或在应用程序的任何其他部分中，包括在后台任务中。

事件回调

当服务器向客户端发出事件时，它可以选择提供回调函数，作为服务器处理事件的确认方式调用。虽然这完全由服务器管理，但客户端可以提供要传递给服务器设置的回调函数的返回值列表。只需从处理函数返回所需的值即可实现：

@sio.event
def my_event(sid, data):
    # handle the message
    return "OK", 123

同样，客户端可以请求在服务器处理完事件后调用回调函数。该socketio.Server.emit()方法有一个可选callback参数，可以设置为可调用。如果给出了此参数，则在服务器处理完事件后将调用callable，并且服务器处理程序返回的任何值都将作为参数传递给此函数。
